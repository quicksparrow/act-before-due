# Security policy

## Reporting a vulnerability

When the public repository is available, use a private GitHub security advisory to report a suspected vulnerability. Do not publish exploit details or private user information in a public issue.

For ordinary bugs that do not expose data or create a security risk, open a regular GitHub issue.

## Security model

ActBeforeDue is local-first. Plans are stored in the user's browser and are not sent to an ActBeforeDue backend. The project intentionally contains no private API keys or server credentials.

The project does not claim that browser storage is encrypted. Anyone with access to an unlocked device or browser profile may be able to view local plans or exported files. Users are instructed never to store sensitive information and to protect their backup files.
